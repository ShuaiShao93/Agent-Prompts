# Guidelines

- Please be cautious about the comments you add in the code. No need to add too many unnecessary details, and make sure the code and the comments are generic enough that they are not specific to the task you are asked to execute.
- Whenever you work on another repo, follow the AGENTS.md or CLAUDE.md in that repo.
- After a PR is opened, check GHA workflows to confirm the non-test checks can pass.
- When updating AGENTS.md or CLAUDE.md file, make sure each bullet is as short as possible.
- Whenever you find AGENTS.md or CLAUDE.md or any skill is outdated or inaccurate, propose to update it
- Whenever you find it take too long to finish a task and you need to do lots of search/experiment to find the solution, propose to make it a skill
- Avoid too many details specific to a particular version or a particular environment in AGENTS.md or CLAUDE.md or skills
